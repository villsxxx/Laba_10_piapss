# Laba_10_piapss
Лабораторная работа 10 : Моделирование информационной системы университета

Этот проект представляет собой простую модель информационной системы университета, реализованную на Java. Система моделирует взаимодействие между сотрудниками, факультетами, институтами, курсами и проектами.

Основные компоненты
University — главная структура, содержащая факультеты.
Faculty — содержит декана и институты.
Institute — содержит научных сотрудников.
Employee — абстрактный класс для всех сотрудников (декан, преподаватель, научный сотрудник).
Participation — ассоциативный класс, связывающий сотрудников с проектами и отслеживающий количество отработанных часов.

Архитектура системы
Система спроектирована с использованием ключевых принципов ООП:

Наследование: Класс Employee является базовым для Dean, Lecturer и ResearchAssociate.
Композиция: Университет состоит из факультетов, которые, в свою очередь, состоят из институтов.
Ассоциация: Класс Participation связывает ResearchAssociate и Project, позволяя отслеживать вклад сотрудника в проект.


Сценарии использования
1. Добавление нового научного сотрудника
Данный сценарий показывает, как декан инициирует процесс добавления нового сотрудника в систему.

equence_diagram.png

Процесс:

Декан отправляет запрос addResearchAssociate в Университет.
Университет находит нужный Факультет.
Факультет передает запрос Институту.
Институт создает объект сотрудника (createAssociate) и подтверждает успешное завершение операции.
Результат возвращается к Декану.



2. Диаграмма прецедентов (Use Case)
На диаграмме показаны роли пользователей и их функциональные возможности.

 use_case_diagram.png

Роли:

Декан (Dean): Управляет бюджетом и факультетами.
Административный сотрудник (AdministrativeEmployee): Регистрирует сотрудников и управляет структурой университета.
Научный сотрудник (ResearchAssociate): Участвует в проектах и просматривает их детали.
Преподаватель (Lecturer): Ведет курсы и просматривает их детали.
